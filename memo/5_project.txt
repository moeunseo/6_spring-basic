※ 면접 단골 질문
▷ 3-Tier Architecture
	애플리케이션을 세 개의 독립된 계층으로 나누어 설계하는 구조.
	각 계층은 특정한 역할과 책임을 가지고 다른 계층과 독립적으로 동작할 수 있다.
	
	1) 프레젠테이션 계층
		HTTP요청(URI형태)을 받고 이 요청을 비지니스 계층으로 전송하는 역할을 한다. (화면이라고 생각)
		사용자로부터 입력을 받고 이를 처리하여 사용자에게 결과를 보여준다.
		웹 페이지와 Controller가 이 계층에 해당!
		
	2) 비지니스 계층
		해당 계층은 비지니스 로직을 처리한다. (개발자들이 개발하는 공간)
		비지니스 로직이란 쉽게 말해 프로세스를 구현하기 위한 모든 로직이라고 생각하면 된다.
		Service가 이 계층에 해당한다.

	3) 퍼시스턴스 계층
		모든 데이터베이스 관련 로직을 처리 (여러 쿼리문 가능)

▶ Mapper
	SQL을 자바 메소드에 매핑하여 데이터베이스와 상호작용할 수 있게 해준다.
	데이터베이스 작업을 추상화하여 코드 내에서 간단하게 호출할 수 있는 메소드로 반환
	select 결과도, 쿼리 내에 들어가는 파라미터도 자동으로 매핑해준다.
		
	mapper 인터페이스 - 자바 메소드와 SQL 매핑을 정의
	maper.xml - SQL 쿼리를 정의 (myBatis 안에 대한 Mapper!)

▶ Service
	애플리케이션의 비지니스 로직을 담당한다. (인터페이스)
	퍼시스턴스 계층과 프레젠테이션 계층 사이의 중간 계층으로, 두 계층 간의 상호 작용을 관리한다.
	하나의 쿼리 또는 여러 개의 쿼리를 실행할 수 있다.
	
▶ Controller
	사용자의 요청을 받아 비지니스 로직을 처리한 후 결과를 사용자에게 응답으로 반환한다.
	HTTP 요청을 매핑하고, 요청을 처리하는 메소드를 정의하며 응답을 반환 (무조건 반환! void가 없음)
	응답은 HTML이 될 수도 있고(@Controller), 데이터(@RestController)가 될 수도 있다.

▶ HTTP 메소드
	- GET
		주소에 데이터를 추가하여 전달하는 방식 > 네이버 검색 시.. url확인!
		보통 쿼리 문자열(query string)에 포함되어 전송되므로, 길이에 제한이 있으며 주소에 데이터가 보이기에 보안 취얀점이 존재한다.
		중요한 데이터 혹은 길이가 긴 데이터는 POST 방식으로 사용하여 요청하는 것이 좋지만 GET방식이 POST방식보다 상대적으로 빠른 전송 방식이다.
		주로 데이터 조회하는데 사용 (SELECT)

	- POST 
		데이터를 별도로 첨부하여 전달하는 방식
		브라우저 히스토리에도 남지 않고 데이터는 쿼리 문자열과는 별도로 전송한다.
		따라서 길이에 제한이 없으며, GET 방식보다 보안성이 높으며 상대적으로 느리다.
		주로 INSERT, UPDATE, DELETE에 사용된다.
		ex) 회원가입 진행 시 데이터가 url에 붙으면 안되기 때문!


▷ 템플릿 엔진
	서버에서 데이터를 받아 HTML에 해당 데이터를 넣어 보여주는 도구
	종류 = thymeleaf(스프링에서 권장), JSP 등

▶ thymeleaf
	의존성을 추가하는 것만으로 별다른 설정 없이 사용할 수 있다.

	- 표현식
	1) ${user.name} - 변수의 값을 표현
	2) #{properties.name} - 속성 파일 값을 표현 (거의 쓸 일 없음)
	3) @{test/test} - URI 표현
	4) *{name} - th:object=${user} - 반복적인 것을 한방에 묶기 위한!
	
	- 문법: 전부 속성이라고 생각!
	1) th:text - 텍스트를 표현할 때 사용
		ex) th:text = "${user.name}"

	2) th:object - 선택한 객체로 지정
		ex) th:object="${user}"
		    th:text="${name}"

	3) ★ th:each - 반복할 때 사용
		ex) th:each="user:${users}"

	4) th:if - 조건이 true일 때 해당 태그 표시	
		ex) th:if="${user.age} > 19"

	5) th:unless - 조건이 false일 때 해당 태그 표시	
		ex) th:unless="${user.age} > 19"

	6) th:href - URI 경로 지정
		ex) th:href="@{/test/test}" > /가 붙은 이유는 앞에 최상위 경로가 붙어있다. localhost:8090/test ~

	7) th:with - 변수 생성
		ex) th:with="name=${user.name}"
